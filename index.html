Are We Being Fooled by Machine Learning when Detecting Code Smells?

Research Paper / Open Source

Code smells indicate poor implementation choices that may degrade the software quality. Hence, studies argue that code smells need to be carefully detected to avoid software degradation. In this context, machine learning techniques havebeen used to detect code smells. However, these techniques do not consider that the identification of smells into code just suggest that it may lead to software degradation; this does not mean that the code is harmful to the software. By harmful code, we mean acode that already had bugs in the past and contains smells, i. e., it is still bug-proneness. This paper presents a study to investigate the presence of harmful code and how effective are machine learning techniques to detect it. We performed our study with 28 smell types, eight open-source projects, and 7144 bugs. Theresults indicate that machine learning techniques detect a high number of smells. But, only a reduced number of these smells are harmful. Moreover, those techniques present high effectiveness to recognize harmful code. Therefore, developers should focus on detecting harmful code instead of smells.
